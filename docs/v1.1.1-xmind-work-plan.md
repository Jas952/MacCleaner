# MacCleaner v1.1.1 — полный план работ по обновлённому XMind

Источник требований: `MacCleaner.xmind`, ветка `Правки для некст апдейтов → v1.1 → v1.1.1`.

Статусы XMind `Требуется повтор`, `Требуется повтор, НО` и `Требуется проверка, НО` считаются незакрытыми. Текст после `НО` — дополнительное обязательное требование. Пункт закрывается только после проверки кода, сборки и визуальной/runtime-проверки соответствующего сценария.

## 1. Optimize

### OPT-01 — заполнение и типографика экранов

- Перепроверить все шесть инструментов Optimize и режим Optimization.
- Уменьшить чрезмерные вертикальные отступы и негармоничные переносы.
- Полезное содержимое должно занимать не менее 40% доступной области при штатном размере окна.
- Заголовок, пояснение, статус, основное действие и результат должны образовывать одну читаемую вертикальную композицию.
- Проверка: визуальный проход RAM, Disk Junk, Refresh, DNS Cache, Safe Delete, Startup и Optimization в состояниях idle/running/result.

### OPT-02 — выравнивание таблиц

- Для всех таблиц Optimize сверить ширины и alignment заголовков с соответствующими ячейками.
- Колонки не должны прыгать при изменении чисел, статусов или появлении кнопки.
- Проверка: Startup и все списки результатов на коротких и длинных строках.

### OPT-03 — контраст кнопок

- Найти все текстовые кнопки Optimize и проверить normal/hover/pressed/disabled в светлой и тёмной темах.
- Текст, фон и граница должны оставаться различимыми; невидимых кликабельных областей быть не должно.
- Проверка: визуальная матрица состояний плюс accessibility labels.

### OPT-04 — Startup

- Убрать серую/чужеродную нижнюю панель, привести её к палитре экрана.
- Выровнять текст выбора, итоговую строку и `Disable Selected`.
- Исключить изменение высоты панели при выборе и очистке выбора.
- Проверка: без выбора, с одним и несколькими элементами, scanning и mutating.

### OPT-05 — стабильная карточка DNS

- Заголовок DNS и соседние метрики не должны смещаться при переходе `Ready → Cleared`.
- Зарезервировать одинаковую высоту для вторичного статуса во всех фазах.
- Проверка: записать координаты/скриншоты до и после очистки.

### OPT-06 — расположение статусных кругов

- На каждом экране Optimize увеличить статусный круг и сместить его левее/к центру, чтобы он не прижимался к углу.
- Размер и положение должны быть единообразными для всех инструментов.
- Проверка: наложение скриншотов шести инструментов.

### OPT-07 — полный сброс при выходе

- После скана, отчёта и успешной очистки выход из Optimize должен вернуть раздел в исходное состояние.
- Сбрасываются выбранный инструмент, результаты, логи, прогресс, сообщения, выбор файлов и фазы; активная операция не прерывается неявно.
- Проверка: отдельный сценарий для каждого инструмента и Optimization, повторный вход должен показывать стартовый экран.

### OPT-08 — анимации переходов

- Убрать «шторки», slide/move и резкие scale-переходы внутри Optimize.
- Использовать умеренный fade из полупрозрачного состояния с единым timing.
- Проверка: переключение grid/tool/mode/result без бокового сдвига.

## 2. Общие требования ко всему приложению

### APP-01 — единый кастомный overlay вместо popover/dropdown

- Информационные popover и выпадающие панели, открываемые кнопками, заменить на кастомный overlay в стиле Process Detail.
- Не использовать визуально нативные sheet/popover как итоговый дизайн.
- Одинаковый размер оболочки; длинное содержимое прокручивается.
- Обязательная кнопка закрытия в виде значка `xmark`.
- Без закруглённых углов.
- Заголовок и кнопка закрытия имеют безопасные отступы от границ.
- Содержимое получает явные секции/карточки и не оставляет чрезмерного пустого пространства.
- Проверка: инвентаризация всех `popover`, `Menu`, информационных `sheet` и аналогичных контролов во всём проекте; каждый случай либо мигрирован, либо документирован как настоящий системный picker/context menu.

### APP-02 — убрать плашки «частичные результаты»

- Найти все пользовательские баннеры/плашки о partial/limited output и убрать их со всех экранов.
- Техническое ограничение можно сохранить компактно в статусе/отчёте, но не отдельной навязчивой плашкой.
- Проверка: глобальный поиск строк и runtime-сценарии ограниченного скана.

### APP-03 — глобальный контраст кнопок

- Провести ревизию всех текстовых кнопок приложения в обеих темах и всех состояниях.
- Создать/использовать единые button styles вместо локальных сочетаний, приводящих к белому тексту на белом фоне.
- Проверка: визуальная ревизия каждого раздела и поиск `.buttonStyle`, `.foregroundStyle`, `.background`.

## 3. Storage

### STO-01 — навигация по папке кликом по заголовку

- Название папки в списке Disk Map должно быть кликабельным и открывать её дочерний уровень.
- Chevron остаётся дополнительной, но не единственной зоной навигации.
- Проверка: папка с детьми, пустая папка, файл, защищённый узел.

### STO-02 — стабильная ширина панели Delete

- Появление выбора и Delete не должно расширять или сдвигать toolbar/контент.
- Кнопка компактная, но сохраняет достаточную hit-area.
- Проверка: сравнение геометрии до выбора, с одним и несколькими выбранными узлами, во время удаления.

### STO-03 — устранить лаг Uninstaller

- Профилировать переход между приложениями и определить источник задержки.
- Убрать синхронный filesystem/process I/O из body/main actor; кэшировать повторно используемые метаданные и отменять устаревшую загрузку.
- Проверка: серия из 20 быстрых переключений, отсутствие hitch по UI и лишних повторных сканов.

### STO-04 — Exact Duplicates: нижняя панель и контраст

- Убрать серый/сливающийся фон и обеспечить видимость всех текстовых кнопок.
- Выровнять summary и действия по одной базовой линии.
- Переименовать `Move to Trash` в `Delete`.
- Сместить Delete левее так, чтобы правая граница совпадала с правой границей карточек результатов.
- Панель не должна менять высоту или ширину при выборе/очистке.
- Проверка: light/dark, no selection/selected/cleaning/result.

### STO-05 — корректность Exact Duplicates

- Подтвердить pipeline size → sample → full SHA-256 и защиту hard links/cloud placeholders.
- Непосредственно перед удалением повторно проверить содержимое и наличие сохраняемой копии.
- После Trash немедленно удалить исчезнувшие элементы из модели; повторный скан не должен возвращать отсутствующие пути.
- Добавить тесты для одинаковых файлов, файлов одного размера с разным содержимым, изменённого после скана файла, hard links, неудачного удаления и повторного скана.
- Проверка: unit/integration tests во временной директории и ручная проверка Finder Trash.

### STO-06 — Specialized Analysis: новая композиция карточек

- Во всех карточках сначала круг/иконка, под ним отдельная кнопка действия, ниже описание.
- Выравнивание и высоты карточек одинаковые, текст переносится гармонично.
- Проверка: все четыре Specialized Analysis карточки при минимальной и широкой ширине окна.

### STO-07 — сброс Storage при выходе

- После скана, отчёта и очистки каждый инструмент Storage при выходе сбрасывается к стартовому экрану.
- Сбрасываются результаты, выбор, баннеры, прогресс и текущий tool; активная операция не уничтожается молча.
- Покрыть Advisor, Exact Duplicates, Similar Photos, Cloud, Uninstaller, Disk Map, Large Files и Junk Files.
- Проверка: отдельный сценарий каждого инструмента и повторный вход.

### STO-08 — положение Cleanup Report

- Кнопка только с иконкой располагается непосредственно после `SSD & Storage` в той же строке.
- Она находится выше текста `Analyze, visualize, and clean your files`, а не после всего блока заголовка.
- Проверка: alignment заголовка и иконки, tooltip/accessibility label, отсутствие сдвига subtitle.

## 4. Проверка обновлений

### UPD-01 — корректный статус проверки

- Устранить постоянный `Unable to check for updates`.
- Различать: updater ещё не готов, нет сети, неверный appcast, актуальная версия и доступное обновление.
- Не показывать failure до реальной ручной/фоновой попытки.
- Проверка: валидный appcast, offline, timeout, up-to-date, available update.

### UPD-02 — кастомное окно по `(i)` и обновлённые тексты

- Открывать кастомный overlay по правилам APP-01, не нативный popover/sheet.
- Успешный результат ручной проверки — явно зелёный и понятный (`MacCleaner is up to date`).
- Toggle переименовать в `Automatically update`.
- Постоянное зелёное сообщение убрать: после ручной проверки оно показывается несколько секунд внутри области кнопки, затем возвращается `Check for Updates`.
- Выполнить дополнительную компоновку ниже статусной области без лишней пустоты.
- Проверка: idle/checking/success/available/failure и автоматическое исчезновение success.

### UPD-03 — фоновая ручная проверка

- Ручная кнопка запускает проверку без системных всплывающих окон.
- Во время запроса кнопка превращается в progress indicator и блокирует повторный запуск.
- Результат отображается в том же overlay.
- Проверка: отсутствие окна Sparkle во всех результатах.

## 5. Dashboard / Processes

### PROC-01 — агрегация процессов и раскрытие экземпляров

- Объединять связанные экземпляры приложения в одну строку с количеством.
- Агрегировать CPU, memory, disk и другие отображаемые метрики по всем экземплярам, а не только memory/disk.
- При раскрытии показывать каждый реальный процесс ровно один раз с PID и индивидуальными метриками.
- Сумма дочерних строк должна совпадать с агрегированной строкой с учётом понятной политики округления.
- `Show Agents` не должен создавать дубликаты; agent/index attribution должен сохраняться для всех экземпляров.
- Действие Quit/Kill должно явно работать с выбранным экземпляром либо запрашивать подтверждение для группы, но не использовать случайный representative PID.
- Проверка: приложение с несколькими helpers/agents; автоматический тест агрегации и ручное сравнение с исходным process snapshot.

- Добавить полный набор Exact Duplicates из STO-05.

## 7. Финальная ревизия всего приложения

### Статический аудит

- Повторно найти `popover`, `Menu`, `sheet`, `transition(.move`, partial/limited notices и локальные button styles.
- Проверить, что каждый найденный случай соответствует APP-01/APP-02/APP-03 либо является допустимым системным диалогом.
- Сверить каждый ID этого документа с конкретным diff и тестом/runtime-доказательством.

### Визуальный проход

- Открыть Dashboard, Processes, Fans, Disk, Optimize, Storage, Desktop, Pake Apps, Agents, Library, Tools и About.
- Проверить light/dark, штатный и минимальный размер окна.
- На каждом экране проверить заголовки, пустое состояние, loading, result, error, кнопки и overlay.
- Для затронутых сценариев сохранить скриншоты до/после или иное воспроизводимое визуальное доказательство.

### Функциональный проход

- Выполнить безопасные сканы, отмену, повторный вход, выбор/очистку выбора и отчёты.
- Проверить обновления в фоне и временный success-status.
- Проверить Process aggregation/expansion на реальном snapshot.
- Проверить Exact Duplicates во временной тестовой директории и через Finder Trash.

### Условие завершения

Работа считается завершённой только когда OPT-01…08, APP-01…03, STO-01…08, UPD-01…03 и PROC-01 имеют прямое доказательство выполнения. Успешная сборка сама по себе не закрывает визуальные или поведенческие требования.
