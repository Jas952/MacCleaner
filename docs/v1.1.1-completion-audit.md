## Runtime-ревизия

Проверены отдельными запусками при размере окна 1300×792:

- Dashboard — агрегированные группы и суммарные метрики;
- Processes — количество экземпляров, group action и колонки;
- Optimize Optimization — композиция, размер/положение круга, заполненность;
- Optimize Startup — toolbar, таблица и нижняя панель;
- Storage home — Cleanup Report и Specialized Analysis;
- Exact Duplicates idle — toolbar и контраст действий;
- Updates overlay — геометрия и контраст;
- Fans, Disk, Desktop, Pake Apps, Agents, Library, Tools и About — отсутствие общих layout/contrast регрессий.

Во время ревизии были обнаружены и исправлены дополнительные дефекты: Dashboard первоначально не агрегировал процессы, общий ключ `com` сливал разные executables, Updates/Startup/Duplicates содержали белый текст на белом фоне, а Library всё ещё использовал скрытые AppKit NSMenu.

Дополнительная контрастная ревизия заменила все оставшиеся системные `.bordered` и `.borderedProminent` в пользовательских Views на общие `AppPrimaryButtonStyle` и `AppSecondaryButtonStyle`. Primary-кнопки всегда имеют белый текст на явной тематической заливке; secondary-кнопки — тёмный текст, светлую semantic-заливку и видимую границу. Runtime-проверены Cleanup Advisor, Exact Duplicates, Similar Photos и Cloud Reclaim.

Повторная проверка пользовательских скриншотов выявила отдельный источник белого-на-белом: системные segmented `Picker` скрывали невыбранные сегменты. Все `.pickerStyle(.segmented)` удалены из Views и заменены на `AppSegmentedControl` с явными selected/unselected цветами. Runtime подтверждает одновременную видимость `Efficient` и `Thorough` в Exact Duplicates и Similar Photos; тот же компонент используется в Cloud Reclaim и Tools.

## Обновлённая архитектура Storage

По уточнённой схеме владельца главная Storage сохраняет прежний визуальный язык карточек: четыре Core cleanup-блока размещены сеткой 2×2 слева, а `Complete Analysis` использует тот же rounded-блок справа и заканчивается на одной линии с Disk Map. Отдельные заголовки Core cleanup/Specialized analysis и внутренняя кнопка Open Analysis удалены. Верхняя область навигационных блоков зафиксирована; независимо прокручивается только прежний Cleanup Intelligence report ниже неё.

Новый `Complete Analysis` последовательно запускает Cleanup Advisor, Exact Duplicates, Similar Photos и Cloud Reclaim одной кнопкой `Scan`, чтобы тяжёлые операции не конкурировали за диск и CPU. Внутри круг расположен по центру, Scan — непосредственно под ним, затем идёт подробная информация; декоративные рамки удалены. После завершения формируется один длинный прокручиваемый отчёт: executive summary и подробные категории со всеми рекомендациями, группами, путями, размерами, разрешениями фотографий, статусами keeper/copy и cloud metadata.

Зависание большого отчёта исправлено: категорийные контейнеры переведены с eager `VStack` на `LazyVStack`, удалены копии `Array(enumerated())` для крупных групп и массовый `.textSelection(.enabled)` на тысячах строк. Повторный запуск приложения после принудительного завершения проверен.

Cleanup Intelligence на главной дополнительно уплотнён: общая прокрутка отчёта удалена, повторяющиеся пояснения Categories/Recurring Paths сокращены, а локальные ScrollView оставлены только внутри этих двух списков при переполнении. Метрики Stable Reclaim, Rebuildable Cache, 30-day Reclaim и Tracked Targets оформлены отдельными белыми блоками с рамкой и лёгкой тенью. Верхние навигационные карточки увеличены до 108 pt; Complete Analysis имеет 230 pt и сохраняет выравнивание по нижней границе второй строки.

Стартовые экраны Storage унифицированы одним порядком: круг → заголовок → кнопка → поясняющий текст. Удалён прежний флаг `actionImmediatelyBelowIcon`, из-за которого Core и Specialized экраны имели разную геометрию, а кнопки Junk/Large/Disk оказывались после фиксированного текстового блока. Runtime-проверены Junk Files, Large Files, Disk Map, Cleanup Advisor, Exact Duplicates, Similar Photos, Cloud Reclaim и Complete Analysis.
